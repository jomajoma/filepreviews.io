You authenticate to the FilePreviews API by providing your application's API Key in the request. You can create an application for a new API Key or get an API Key from an existing application.

If you're planning on making requests from a client-side application with your API Key, you should set your application's Origin whitelist setting which authorizes specific origin hostnames to make requests with your API Key. Remember, that should you never expose your Secret Key in any public website's client-side code.

We allow unauthenticated requests to some endpoints. These have a restrictive throttle of 20 requests per hour using the IP address of the incoming request. Throttled requests will return a 429 Too Many Requests response.

Authentication to the API occurs via a custom HTTP X-API-KEY request header or via the api_key query string parameter.

All API requests must be made over HTTPS.
