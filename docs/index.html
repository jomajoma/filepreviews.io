The FilePreviews.io service is composed of a very simple API. There are two endpoints that can be used to request a preview, which happens asynchronously. They both work similarly but are useful for different scenarios and is organized around REST.

We support cross-origin resource sharing to allow you to interact securely with your API from a client-side web application. JSON will be returned in all responses from the API, including errors.

Libraries

Libraries are available in several languages.

Versioning

Currently, the FilePreviews API is versioned with a URL segment, such as /v1/. The current production version of the API is Version 1, so your calls should use /v1/ as the version segment.

API Endpoint

https://api.filepreviews.io/v1/
