GET /v1/

Useful endpoint for client-side applications. Returned response can be used for polling until results are available.

Example Request

{% highlight bash %} $ curl --get https://api.filepreviews.io/v1/ \ --data "url=https://www.filepicker.io/api/file/QLOxaa2ETEOMSQ0kx4u1" \ --data "size=320>" \ --data "format=png" \ --data "metadata=ocr,checksum" {% endhighlight %}

Example Response

{% highlight json %} { "metadata_url": "https://s3.amazonaws.com/demo.filepreviews.io/ceebd7f694e2e7e4645db44e6f052d38d83d81d6bd4be73e4423016bcdf96087/metadata.json", "preview_url": "https://s3.amazonaws.com/demo.filepreviews.io/ceebd7f694e2e7e4645db44e6f052d38d83d81d6bd4be73e4423016bcdf96087/QLOxaa2ETEOMSQ0kx4u1_320>_1.png" } {% endhighlight %}
Name Required Description
url

The direct publicly accessible URL of the file to generate previews. This can be a temporary signed Amazon S3/Cloudfront URL or contain basic authentication credentials.
api_key

An application's API Key. It's recommended that you specify this so you can track your usage, bypass anonymous usage throttling, and upload your assets to a specified destination.
size

The size of the extracted thumbnails, in pixels. Expressed as widthxheight. Specifying only the width(e.g "300") will generate a thumbnail with a fixed width and varying height. You can also only shrink larger images to fit into the give size and never enlarge(e.g "300x400>").
format

The format of the extracted thumbnails. Supported values are "jpg", "jpeg", and "png".
metadata

Comma-separated values of addititional metadata to extract. Possible values are "ocr", "psd", "exif", "ocr", "checksum", "multimedia", and "all".
pages

The document pages that you want to extract thumbnails for. You can specify a range or individual pages separated by commas. Example: "1-3" or "1,3,5". Default value is "1".
